The \mbox{\hyperlink{namespaceArgParser}{Arg\+Parser}} library allows for parsing command line arguments through a user-\/defined class which enables easy access to parsed values.\hypertarget{md_README_autotoc_md1}{}\doxysection{key concepts}\label{md_README_autotoc_md1}

\begin{DoxyItemize}
\item The library relies on C\# reflection. Which allows accessing parsed values with correct type and through pre-\/defined field.
\item Possibility of custom Options or Arguments by inheriting from {\ttfamily Option\+Base\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}} or {\ttfamily Argument\+Base\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}}
\item Short option vs long option
\begin{DoxyItemize}
\item short option (uses {\ttfamily -\/} with parameter name) has name consisting of 1 char only
\item long option (uses {\ttfamily -\/-\/} with parameter name) has name consisting of at least 2 chars
\end{DoxyItemize}
\item Parsed option/arg types are restricted using inheritance\+: descendants of {\ttfamily Argument\+Base\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}} and {\ttfamily Option\+Base\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}}. Which limits possible types that could be parsed.
\end{DoxyItemize}\hypertarget{md_README_autotoc_md2}{}\doxysection{use cases}\label{md_README_autotoc_md2}

\begin{DoxyEnumerate}
\item The user will define the accepted arguments as fields in a class inherited from {\ttfamily Parser\+Base}.
\begin{DoxyItemize}
\item The fields defined in the parser should themselves inherit from {\ttfamily Argument\+Base\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}} or {\ttfamily Option\+Base\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}}, however, the pre-\/defined classes such as {\ttfamily Int\+Option} and {\ttfamily Int\+Argument} should cover most basic use-\/cases (see Simple example).
\item Difference between option and argument is that options are prefixed by {\ttfamily -\/} or {\ttfamily -\/-\/} and arguments are plain arguments, therefore without delimiter.
\item {\ttfamily Parameter\+Accept} describes number of parsed parameters. By default min and max number of parsed params is set to 1.
\item {\ttfamily Get\+Argument\+Order} is used to define order of plain arguments. It should contain references to the parsed fields of the Parser.
\item {\itshape Only one} Argument of a parser can have variable acceptable count of arguments ({\ttfamily Parameter\+Accept} where its minimum and maximum value are not the same). Otherwise {\ttfamily Parser\+Code\+Exception} is thrown.
\end{DoxyItemize}
\item User calls {\ttfamily Parser\+Base.\+Parse()}. The parser will identify all the necesarry fields it contains and fill them with the parsed values.
\begin{DoxyItemize}
\item Throws {\ttfamily Parse\+Exception} if parsed arguments don\textquotesingle{}t satisfy declared option fields
\item Throws {\ttfamily Parser\+Code\+Exception} if the class doesn\textquotesingle{}t conform to the parser requirements
\begin{DoxyItemize}
\item Fields not inheriting from predefined classes (descendants of {\ttfamily Option\+Base\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}} or {\ttfamily Argument\+Base\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}})
\item Defining argument fields without overriding {\ttfamily Get\+Argument\+Order}
\end{DoxyItemize}
\end{DoxyItemize}
\item User obtains parsed values from the fields by calling a method {\ttfamily Get\+Value(int idx)} on them.
\end{DoxyEnumerate}\hypertarget{md_README_autotoc_md3}{}\doxysection{Simple example}\label{md_README_autotoc_md3}
